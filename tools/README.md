# כלי אריזה — EXE ייחודי עם משחק סגור בפנים

`seal-game.mjs` אורז קובץ משחק (ZIP) לתוך ה-EXE הנייד הגנרי ומייצר **EXE ייחודי**
שטוען את המשחק אוטומטית בהפעלה — בלי בורר קבצים, לא ניתן להחלפה. אין צורך לבנות
את ה-EXE מחדש לכל משחק: המשחק פשוט מודבק לסוף קובץ ה-EXE הגנרי (קבצי EXE מתעלמים
מבתים נגררים אחרי מבנה ה-PE).

## דרישות
- Node.js
- ה-EXE הגנרי: `TriviaEngine-Portable.exe` (מעמוד ההורדות / הרִיליס).
- קובץ המשחק כ-ZIP (עם `data.json` + תיקיית הנכסים).

## שימוש
```bash
node tools/seal-game.mjs \
  --exe TriviaEngine-Portable.exe \   # ה-EXE הגנרי
  --game mygame.zip \                 # קובץ המשחק (ZIP)
  --out MyGame.exe \                  # ה-EXE הייחודי שייווצר
  --room ABC123 \                     # קוד חדר לטלפונים (בלי = אין טלפונים)
  --clickers true \                   # לאפשר שלטים (ברירת מחדל: כן)
  --phones true \                     # לאפשר טלפונים (ברירת מחדל: כן אם יש room)
  --limit 200 \                       # מגבלת משתתפים (בלי = כמו ב-JSON)
  --name "שם המשחק"
```
או דרך npm: `npm run seal -- --exe ... --game ... --out ...`

## מה מוטבע
- **המשחק** (ה-ZIP) — נטען אוטומטית, לא ניתן להחליף.
- **מקורות הצבעה מותרים**: שלטים (RF317) ו/או טלפונים — לפי `--clickers`/`--phones`.
  מסך "איך משחקים?" יציג רק את האפשרויות המותרות (ואם צריך, גם "שלטים + טלפונים").
- **קוד חדר** (`--room`) — כדי שהטלפונים יתחברו אונליין.
- **מגבלת משתתפים** (`--limit`).

## שילוב בשרת
הפעל את הסקריפט מהשרת שיוצר את המשחקים: קלט = קובץ המשחק (+ הפרמטרים),
פלט = ה-EXE הייחודי להורדה. הלוגיקה הטהורה נמצאת ב-`electron/sealPayload.cjs`
(אפשר לייבא אותה ישירות מקוד שרת Node אם מעדיפים endpoint על פני CLI).
